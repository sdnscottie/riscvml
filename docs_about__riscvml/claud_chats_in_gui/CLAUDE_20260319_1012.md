# ANKi-Ref Project Documentation System

**Date:** March 19, 2026  
**Context:** Hardware project organization and documentation workflow using ANKI spaced repetition  
**Projects:** RISCVML, DrApp, ESP32-P4 development, motor control systems  

## Overview

Scott demonstrates an innovative physical-to-digital project documentation system that combines:
- **Physical organization**: Ikea wood cabinet with labeled project boards
- **Digital documentation**: ANKI flashcard system with timestamped references
- **Visual tracking**: Photo attachments showing exact project state
- **Portable workflow**: Boards can be moved to desk for active development

## Physical Organization System

### Cabinet Setup (Ikea Wood Cabinet)
- **Top drawer**: Tools/components storage
- **Upper shelves**: Clean workspace areas  
- **Mid-level shelf**: Active projects with ANKi-Ref labels
- **Bottom shelf**: Cable management and wiring storage

**Benefits:**
- Clear project segregation
- Easy access to hardware components
- Organized workspace for multiple concurrent projects
- Physical-digital workflow integration

## ANKi-Ref Documentation System

### Reference Format
```
ANKi-Ref: [TIMESTAMP]
Example: ANKi-Ref: 20260319-093229
```

**Timestamp Format:** `YYYYMMDD-HHMMSS`
- Year: 2026
- Month: 03 (March)
- Day: 19
- Time: 09:32:29

### Workflow
1. **Label Creation**: Each project board gets unique ANKi-Ref timestamp
2. **Physical Mobility**: Pull labeled board to desk for development
3. **Photo Documentation**: Capture exact project state at timestamp
4. **ANKI Card Creation**: 
   - Reference number as card ID
   - Photo attachment showing circuit/wiring
   - Progress notes and learnings
   - Code snippets or debugging insights
5. **Spaced Repetition**: Review prevents context loss over time
